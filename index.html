Open substrate

The JCS canonicalisation substrate for agentic payments

AlgoVoi authors the open JCS (RFC 8785) canonicalisation discipline and the verifiable receipt formats that let an agent payment be re-derived and checked by anyone — across x402, AP2, A2A and MPP, on one shared substrate.

What the substrate defines

  • action_ref — SHA-256(JCS({agent_id, action_type, scope, timestamp_ms})): a content-addressed reference to a payment action, recomputable from the disclosed fields alone.
  • Compliance receipt — categorical ALLOW / REFER / DENY screening outcome, signed and no-PII.
  • Settlement attestation — binds the on-chain settlement to the same action_ref.
  • RFC 9421 signing base — the canonical signing input for HTTP message signatures across a proxy chain.

Open, and verifiable yourself

The conformance vectors are published under Apache-2.0 and cross-validated byte-identical across eight independent implementations in eight languages. Import the set by digest and confirm your implementation produces the same bytes.

  • Conformance vectors & verifier: algovoi-jcs-conformance-vectors
  • Receipt verifier: algovoi-receipt-verifier
